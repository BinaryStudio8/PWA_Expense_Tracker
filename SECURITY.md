# Security Policy

## Supported Versions

Only the actively maintained versions receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | ✅ Supported        |
| 5.0.x   | ❌ Unsupported      |
| 4.0.x   | ✅ Supported        |
| < 4.0   | ❌ Unsupported      |

If your version is not listed here, update before reporting issues.

---

## Reporting a Vulnerability

If you discover a security vulnerability, report it responsibly and privately.

### How to Report
Email us at:

**security@yourdomain.com**  
_(replace this with your real security email)_

Include the following details:
- Description of the vulnerability  
- Steps to reproduce  
- Expected vs actual behavior  
- Potential impact  
- Any logs, proofs, or screenshots  
- Whether the issue is already public  

### What to Expect
- You will receive an acknowledgement within **48 hours**  
- Status updates at least once every **7 days**  
- If confirmed, we will work with you on remediation  
- Credit will be given to researchers who follow responsible disclosure  

### Responsible Disclosure
Do **not** publish or share vulnerability details until:
- a fix is released, **and**
- users have had reasonable time to update.

We take security seriously. Proper reporting helps keep everyone safe.
